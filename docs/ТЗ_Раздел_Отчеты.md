# ТЗ: Раздел «Отчёты» — INTERNO School App

**Версия:** 1.0
**Дата:** 17 марта 2026
**Основа:** Google Таблица "Таблица отчетов INTERNO School Ташкент 2025-2026"

---

## 1. Общее описание

Раздел **«Отчёты»** — это интерактивный модуль для отслеживания ежедневных, еженедельных и ежемесячных KPI менеджеров отдела продаж. Полностью заменяет текущую Google Таблицу, обеспечивая автоматический расчёт конверсий, визуализацию данных и удобный ввод с любого устройства.

---

## 2. Роли и доступ

| Роль | Просмотр | Ввод данных | Установка планов | Редактирование чужих данных |
|------|----------|-------------|------------------|-----------------------------|
| **Владелец (owner)** | Все менеджеры, все филиалы | Нет | Да | Да |
| **РОП (rop)** | Все менеджеры своего филиала | Нет | Да (свой филиал) | Да (свой филиал) |
| **Администратор (admin)** | Все менеджеры | Нет | Нет | Нет |
| **Менеджер по продажам (sales_manager)** | Только свои данные | Да (только свои) | Нет | Нет |
| **Остальные роли** | Нет доступа | Нет | Нет | Нет |

## 3. Структура данных

### 3.1. Планы (`salesReportPlans`)

```
{
  id: string,
  month: "2026-03",          // год-месяц
  branch: "branch_1",        // филиал
  employeeId: number,        // ID менеджера
  employeeName: string,      // Имя менеджера
  plans: {
    leads: 260,              // Кол-во заявок (месячный план)
    conversations: 1300,     // Кол-во разговоров
    signups: 130,            // Кол-во записей
    visited: 104,            // Кол-во пришедших
    sales: 35,               // Кол-во продаж
    revenue: 150000000       // Выручка (сум)
  }
}
```

### 3.2. Ежедневные данные (`salesReportDaily`)

```
{
  id: string,
  date: "2026-03-17",        // конкретная дата
  branch: "branch_1",
  employeeId: number,
  metrics: {
    leads: 12,               // Кол-во заявок за день
    conversations: 25,       // Кол-во разговоров за день
    signups: 4,              // Кол-во записей за день
    visited: 3,              // Кол-во пришедших за день
    sales: 2,                // Кол-во продаж за день
    revenue: 5800000         // Выручка за день (сум)
  }
}
```

### 3.3. Метрики (автоматически рассчитываемые)

| Метрика | Формула | Описание |
|---------|---------|----------|
| **Кол-во заявок** | Сумма по дням | Входящие заявки/лиды |
| **Кол-во разговоров** | Сумма по дням | Состоявшиеся разговоры |
| **Кол-во записей** | Сумма по дням | Записавшиеся на пробный урок |
| **Конверсия в записи** | `записи / разговоры × 100%` | Автоматический расчёт |
| **Кол-во пришедших** | Сумма по дням | Фактически пришли на урок |
| **Конверсия пришедших** | `пришедшие / записи × 100%` | Автоматический расчёт |
| **Кол-во продаж** | Сумма по дням | Совершённые продажи |
| **Конверсия в продажу** | `продажи / пришедшие × 100%` | Автоматический расчёт |
| **Выручка** | Сумма по дням | Денежная сумма в сумах |

## 4. Интерфейс

### 4.1. Верхняя панель фильтров

```
┌──────────────────────────────────────────────────────────────────────┐
│  📊 Отчёты продаж                                                    │
│                                                                      │
│  [Филиал ▼]  [Месяц ▼: Март 2026]  [Менеджер ▼: Все]               │
│                                                                      │
│  Период: [День] [Неделя] [Месяц]                                    │
└──────────────────────────────────────────────────────────────────────┘
```

**Фильтры:**
- **Филиал** — выпадающий список филиалов (для owner/admin — все, для РОП — свой)
- **Месяц** — выбор месяца/года (по умолчанию текущий)
- **Менеджер** — конкретный менеджер или "Все" (менеджер видит только себя)
- **Период** — переключатель День/Неделя/Месяц (влияет на группировку таблицы)

## 5. Функциональные требования

### 5.1. Ввод данных (менеджеры)

1. Менеджер заходит в раздел «Отчёты»
2. Видит таблицу только со своими данными за текущий месяц
3. Кликает на ячейку нужного дня и метрики
4. Вводит число (целое для кол-ва, число для выручки)
5. Данные сохраняются в Firestore мгновенно
6. Все конверсии и итоги пересчитываются автоматически
7. **Менеджер НЕ может:**
   - Редактировать чужие данные
   - Менять данные за прошедшие дни (опционально, настраивается РОП/owner)
   - Устанавливать планы

### 5.2. Установка планов (РОП / Owner)

1. РОП/Owner нажимает кнопку «Настроить планы»
2. Открывается модальное окно:
   ```
   ┌─── Планы на Март 2026 ───────────────────────────────┐
   │                                                       │
   │  Менеджер: [Евгения ▼]                               │
   │                                                       │
   │  Месячный план:                                       │
   │  Заявки:         [260]                                │
   │  Разговоры:      [1300]                               │
   │  Записи:         [130]                                │
   │  Пришедшие:      [104]                                │
   │  Продажи:        [35]                                 │
   │  Выручка:        [150 000 000]                        │
   │                                                       │
   │  ⚙️ Автоделение на недели: [✓]                        │
   │  (месячный план ÷ кол-во недель = недельный план)     │
   │                                                       │
   │  Общий план филиала: [400 000 000]                    │
   │                                                       │
   │           [Отмена]  [Сохранить]                       │
   └───────────────────────────────────────────────────────┘
   ```
3. Недельные планы рассчитываются автоматически: `месячный план ÷ кол-во рабочих недель`

### 5.3. Просмотр отчётов (РОП / Owner / Admin)

1. Видят данные **всех менеджеров** в одной таблице
2. Могут фильтровать по филиалу и менеджеру
3. Видят строку **ИТОГО** по всем менеджерам
4. Видят отклонение от общего плана
5. Могут переключать месяц для просмотра истории
6. Имеют доступ ко всем графикам

### 5.4. Экспорт

- **Экспорт в Excel** — генерация .xlsx файла в формате, идентичном Google Таблице
- **Экспорт в PDF** — отчёт с графиками для печати
- **Поделиться ссылкой** — ссылка на конкретный отчёт (месяц + филиал)

## 6. Автоматические расчёты

### 6.1. Формулы конверсий

```
Конверсия в записи     = Кол-во записей / Кол-во разговоров × 100%
Конверсия пришедших    = Кол-во пришедших / Кол-во записей × 100%
Конверсия в продажу    = Кол-во продаж / Кол-во пришедших × 100%
```

### 6.2. Агрегации

```
Недельный факт  = Σ (дневных значений за неделю, Пн-Сб)
Месячный факт   = Σ (всех недельных фактов)
% выполнения    = Месячный факт / Месячный план × 100%
ИТОГО           = Σ (фактов всех менеджеров)
Отклонение      = Общий план - ИТОГО факт
```

### 6.3. Недельная разбивка

Месяц делится на недели (Пн-Сб):
- Неделя 1: 1-7 число
- Неделя 2: 8-13
- Неделя 3: 14-19
- Неделя 4: 20-25
- Неделя 5: 26-31 (остаток)

Между неделями — промежуточный итог.

## 7. Хранение в Firestore

### 7.1. Коллекции

| Коллекция | Описание | Примерный объём |
|-----------|----------|-----------------|
| `reportPlans` | Месячные планы менеджеров | ~30 документов/месяц (10 менеджеров × 3 филиала) |
| `reportDaily` | Ежедневные данные | ~300 документов/месяц (10 × 30 дней) |

### 7.2. Индексы

- `reportDaily`: составной индекс по `(branch, month, employeeId)`
- `reportPlans`: составной индекс по `(branch, month)`

### 7.3. Оптимизация чтений

- При открытии раздела загружаются данные только за **выбранный месяц** и **выбранный филиал**
- Подписка `onSnapshot` только на отфильтрованные документы
- Максимум чтений при открытии: ~40 документов (10 менеджеров × 30 дней + 10 планов)

## 12. Пример пользовательского сценария

### Сценарий: Менеджер Евгения вносит данные за день

1. Евгения логинится в систему (роль: `sales_manager`)
2. Переходит в раздел «Отчёты»
3. Видит таблицу со своими данными за текущий месяц
4. Кликает на ячейку «Заявки» в столбце «17 марта»
5. Вводит число `12`, нажимает Enter
6. Заполняет остальные метрики: разговоры, записи, пришедшие, продажи, выручка
7. Конверсии рассчитываются автоматически
8. Недельный и месячный итоги обновляются мгновенно
9. РОП видит обновлённые данные в реальном времени

### Сценарий: Owner смотрит месячный отчёт

1. Owner заходит в «Отчёты»
2. Выбирает филиал «Чиланзар» и месяц «Февраль 2026»
3. Видит сводную таблицу всех менеджеров с планом/фактом
4. Обращает внимание на красную ячейку — Шахло выполнила план заявок на 10.8%
5. Переключается на вкладку «Графики» → видит воронку продаж
6. Экспортирует отчёт в Excel для совещания
